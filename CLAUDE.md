# Правила работы с проектом

1. Для точечных задач сначала используй поиск по точному тексту, названию компонента или CSS-классу.
2. Не анализируй весь проект, если задача относится к одному элементу.
3. Не выполняй рефакторинг без прямого запроса.
4. Не меняй соседние элементы, дизайн или логику без необходимости.
5. Не создавай файлы old, backup, copy, new, final и final2.
6. Не редактируй node_modules, dist, build, .git и сгенерированные файлы.
7. Используй минимальное количество изменений.
8. Не устанавливай новые зависимости без прямого запроса.
9. Не запускай npm install для обычных текстовых правок.
10. После изменения указывай точные изменённые файлы.
11. После значимых изменений проверяй сборку.
12. Не удаляй код, если нет уверенности, что он не используется.
13. Не меняй внешний вид и функции сайта во время технической уборки.
14. Не заменяй существующий фреймворк или систему сборки.

## Исключения при обычном анализе

При поиске и анализе точечных задач не обходи эти каталоги и файлы без прямой необходимости:

- `.git/`
- `node_modules/`
- `dist/`
- `build/`
- `.vite/`
- `.cache/`
- `coverage/`
- временные файлы и системные артефакты
- крупные бинарные ассеты, если задача не связана с изображениями

Для этого проекта также считай `js/app.js` сгенерированным legacy-бандлом. Не редактируй его для обычных текстовых, CSS- или точечных интерфейсных правок, если задачу можно решить в статических файлах, данных, CSS или отдельных JS-модулях.
