# Правила работы с проектом

1. Для точечных задач сначала используй поиск по точному тексту, названию компонента или CSS-классу.
2. Не анализируй весь проект, если задача относится к одному элементу.
3. Не выполняй рефакторинг без прямого запроса.
4. Не меняй соседние элементы, дизайн или логику без необходимости.
5. Не создавай файлы old, backup, copy, new, final и final2.
6. Не редактируй node_modules, dist, build, .git и сгенерированные файлы.
7. Используй минимальное количество изменений.
8. Не устанавливай новые зависимости без прямого запроса.
9. Не запускай npm install для обычных текстовых правок.
10. После изменения указывай точные изменённые файлы.
11. После значимых изменений проверяй сборку.
12. Не удаляй код, если нет уверенности, что он не используется.
13. Не меняй внешний вид и функции сайта во время технической уборки.
14. Не заменяй существующий фреймворк или систему сборки.

## Исключения при обычном анализе

При поиске и анализе точечных задач не обходи эти каталоги и файлы без прямой необходимости:

- `.git/`
- `node_modules/`
- `dist/`
- `build/`
- `.vite/`
- `.cache/`
- `coverage/`
- временные файлы и системные артефакты
- крупные бинарные ассеты, если задача не связана с изображениями

Для этого проекта также считай `js/app.js` сгенерированным legacy-бандлом. Не редактируй его для обычных текстовых, CSS- или точечных интерфейсных правок, если задачу можно решить в статических файлах, данных, CSS или отдельных JS-модулях.

## Дизайн-система Клипер.Сити

### Фаза проекта

Сейчас фаза 1 — только Недвижимость. Активные разделы: Застройщики, Новостройки, Готовые ЖК, Для бизнеса. Всё остальное скрыто через `js/behavior/phase1-cleanup.js`.

### Стек

- **CSS**: Tailwind (скомпилированный в `css/style.css`) + кастомные стили в отдельных CSS-файлах
- **JS**: Vanilla JS модули в `js/` — данные в `js/data/`, поведение в `js/behavior/`, страницы в `js/pages/`
- **Шрифт**: Manrope (400, 500, 600, 700, 800)
- **Рендеринг**: React-бандл в `js/app.js` → `<div id="root">`. Все overlay-модули работают через DOM-наблюдение (MutationObserver)

### Цвета

Все новые элементы должны использовать эти цвета — не изобретать свои:

| Роль | Светлая тема | Тёмная тема (переменная) |
|---|---|---|
| Акцент (primary) | `#7c3aed` → `#6d28d9` (violet-500→700 gradient) | то же |
| Фон страницы | `#f8fafc` (slate-50) | `var(--kliper-dark-page)` `#080d18` |
| Фон карточки/поверхности | `#ffffff` | `var(--kliper-dark-surface)` |
| Фон вложенный | `#f1f5f9` (slate-100) | `var(--kliper-dark-surface-2)` |
| Граница | `#e2e8f0` (slate-200) | `var(--kliper-dark-border)` |
| Текст основной | `#0f172a` (slate-950) | `var(--kliper-dark-text)` `#f8fafc` |
| Текст вторичный | `#64748b` (slate-500) | `var(--kliper-dark-muted)` `#a9b6ca` |
| Хедер тёмный | `#030a1d` | — |
| Тень карточки | `0 14px 38px rgba(15,23,42,0.06)` | `var(--kliper-dark-shadow-soft)` |
| Фокус/кольцо | `violet-500/50` | `var(--kliper-dark-focus-ring)` |

### Типографика

- Заголовок секции: `text-[22px] sm:text-2xl font-black tracking-tight`
- Заголовок карточки: `text-lg font-bold` или `font-black`
- Подзаголовок/мета: `text-xs font-bold text-slate-400`
- Тело текста: `text-sm font-medium`
- Метки/бейджи: `text-[11px] font-black uppercase tracking-[0.18em]`
- Кнопки: `text-[13px] font-black`

### Скругления

- Карточки и секции: `rounded-[20px]` — `rounded-[22px]`
- Кнопки/пилы: `rounded-[10px]` — `rounded-xl`
- Бейджи: `rounded-full`
- Инпуты: `rounded-xl`

### Тени

- Карточка: `shadow-[0_14px_38px_rgba(15,23,42,0.06)]`
- Навбар: `shadow-[0_18px_44px_rgba(15,23,42,0.18)]`
- Кнопка при наведении: `hover:-translate-y-0.5 hover:shadow-lg`
- Модалка/сайдбар: `shadow-[28px_0_90px_rgba(15,23,42,0.22)]`

### Анимации

- Переходы: `transition` (200ms по умолчанию Tailwind)
- Hover кнопок: `hover:-translate-y-0.5`
- Не использовать длительные анимации (>400ms), не добавлять auto-play анимации

### Паттерны UI

- **Карточки**: округлые, с фоновым фото и градиентным overlay `bg-gradient-to-t from-black/70`, бейдж «профиль компании» в левом верхнем углу, аватар компании в правом верхнем
- **Истории**: круглые аватары с gradient-ring (violet), в горизонтальном скролле
- **Фильтры**: pill-кнопки в горизонтальной полосе с `overflow-x-auto`, активная — violet gradient, неактивная — прозрачная
- **Счётчики**: иконка + число, маленький `text-xs`, на карточке внизу (лайки ♡, уведомления △, комменты ◇)
- **Тёмная тема**: класс `.kliper-dark-theme` на `<html>` и `<body>`, переключается через `localStorage('kliper-theme')`

### Адаптивность

- Mobile-first: базовые стили для мобильных, `sm:` / `md:` / `xl:` для расширения
- Точки перелома: `sm:640px`, `md:768px`, `xl:1280px`
- На мобильных: 1 колонка карточек, скроллируемые фильтры
- На десктопе: 2-3 колонки карточек, фильтры в строку

### Обязательная проверка

После любого визуального изменения (CSS, layout, компоненты) проверять **три viewport'а**:

1. **Desktop** (1280x800) — основной вид
2. **Mobile** (375x812) — через `preview_resize preset:mobile`
3. **Тёмная тема** — через `preview_resize colorScheme:dark`

Не считать задачу выполненной, пока все три не проверены. Если мобильная версия ломается — чинить сразу, не откладывать.

Критерии мобильной версии:
- Текст читаемый без зума (min `14px` для основного текста, `11px` для меток)
- Кнопки и интерактивные элементы min `44x44px` touch target
- Нет горизонтального скролла страницы (только внутри скроллируемых полос фильтров)
- Карточки в 1 колонку, без обрезания контента
- Фильтры скроллируемые горизонтально, не схлопываются

## Стандарты кода

### Структура файлов

```
js/
  data/          — конфигурация и данные (JSON-подобные объекты на window.*)
  behavior/      — модули поведения (DOM-манипуляции, темы, фильтры)
  pages/         — логика конкретных страниц
  catalog/       — логика каталога
  filters/       — фильтры
  shared/        — общие утилиты
css/
  style.css      — главный (Tailwind-compiled, не редактировать вручную)
  *.css          — дополнительные кастомные стили
```

### Правила JS-модулей

1. Каждый модуль — IIFE: `(function moduleName() { 'use strict'; ... })();`
2. Для взаимодействия с React-бандлом: MutationObserver на `document.body`, не таймеры
3. Не модифицировать React-состояние напрямую — только через клик/событие на оригинальные DOM-элементы
4. Глобальные данные — через `window.KLIPER_*` (site-config, navigation, contacts, content-labels)
5. Для визуального скрытия элементов, которые должны оставаться кликабельными, использовать `sr-only` паттерн (position:absolute, clip:rect), не `display:none`
6. Версионирование подключений: `?v=описание-версии-N`
7. Не использовать ES6+ синтаксис (arrow functions, const/let, template literals) — бандл legacy, совместимость с ES5

### Правила CSS

1. Новые стили — в отдельные CSS-файлы, не в `style.css`
2. Использовать существующие `--kliper-dark-*` переменные для тёмной темы
3. Селекторы через `.kliper-*` namespace, чтобы не конфликтовать с Tailwind
4. Не использовать `!important` без крайней необходимости (конфликт с Tailwind)
5. Поддерживать тёмную тему: `.kliper-dark-theme .kliper-*` селекторы

### Именование

- CSS-классы: `kliper-{компонент}-{модификатор}` (kebab-case)
- JS-модули: `{область}-{действие}.js` (kebab-case)
- Данные: `window.KLIPER_{НАЗВАНИЕ}` (UPPER_SNAKE_CASE)
- Файлы: описательные имена, без сокращений
